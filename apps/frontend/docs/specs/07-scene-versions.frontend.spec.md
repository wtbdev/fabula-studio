# Frontend Feature Spec 07：场次版本历史

## 1. 功能名称

场次版本历史

## 2. 功能目标

为剧本场次提供历史版本查看和恢复能力。用户在编辑或 AI 重新生成场次后，可以回看过去保存过的内容，并在需要时恢复某个历史版本。

该功能用于降低用户编辑风险，让 AI 重写和手动修改更安全。

---

## 3. 功能范围

### 本阶段实现

1. 在编辑器中提供“历史版本”入口；
2. 展示当前场次的版本列表；
3. 查看某个版本内容；
4. 恢复某个历史版本；
5. 区分版本来源：手动保存、AI 重生成、版本恢复；
6. 恢复前二次确认。

### 本阶段不实现

1. 两个版本 diff 高亮；
2. 自定义版本标题；
3. 项目级完整版本历史；
4. 自动合并版本；
5. 版本批量删除。

---

## 4. 页面入口

项目编辑页：

```txt
/projects/:id/edit
```

按钮位置：

1. 编辑器顶部工具栏；
2. 或保存按钮旁边。

按钮文案：

```txt
历史版本
```

---

## 5. 交互流程

### 5.1 打开版本历史

1. 用户选中某个场次；
2. 点击“历史版本”；
3. 打开右侧抽屉或弹窗；
4. 前端请求当前场次的版本列表；
5. 展示版本列表。

---

### 5.2 查看版本

1. 用户点击某个版本；
2. 右侧或下方展示该版本内容；
3. 当前选中版本高亮。

---

### 5.3 恢复版本

1. 用户点击“恢复此版本”；
2. 弹出确认框：

```txt
确定将当前场次恢复到版本 v3 吗？当前未保存内容可能被覆盖。
```

3. 用户确认；
4. 调用恢复接口；
5. 恢复成功后更新编辑器内容；
6. 关闭抽屉或保持打开；
7. 显示提示：

```txt
已恢复历史版本
```

---

## 6. UI 设计

建议使用右侧抽屉：

```txt
┌──────────────────────────┐
│ 历史版本                  │
├──────────────────────────┤
│ v5 手动保存  18:30        │
│ v4 AI 重生成 18:12        │
│ v3 手动保存  17:50        │
├──────────────────────────┤
│ 版本内容预览               │
│ ...                      │
│ [恢复此版本]               │
└──────────────────────────┘
```

版本来源标签：

| source | 展示文案 |
|---|---|
| manual_save | 手动保存 |
| ai_regenerate | AI 重生成 |
| restore | 版本恢复 |

```http
GET /api/scenes/:sceneId/versions
GET /api/scene-versions/:versionId

```

---

## 9. 状态设计

```ts
interface VersionState {
  visible: boolean
  loading: boolean
  versions: SceneVersion[]
  currentVersionId: string | null
  currentVersionContent: string
  restoring: boolean
}
```

---

## 10. 版本创建规则

前端需要理解以下规则：

1. 用户保存场次时，后端可自动创建版本；
2. 用户应用 AI 重新生成内容时，后端创建版本；
3. 用户恢复历史版本时，后端创建一个新的 restore 版本；
4. 前端不直接创建版本，只负责触发保存、重生成和恢复。

---

## 11. 错误处理

| 场景 | 前端提示 |
|---|---|
| 无历史版本 | 当前场次暂无历史版本 |
| 获取失败 | 历史版本加载失败 |
| 恢复失败 | 版本恢复失败，请重试 |
| 未保存内容 | 当前内容尚未保存，恢复后可能被覆盖 |

---

## 12. 验收标准

1. 编辑器有历史版本入口；
2. 点击后可以打开版本历史抽屉；
3. 可以看到当前场次的版本列表；
4. 版本列表能显示版本号、来源和时间；
5. 点击版本能查看内容；
6. 点击恢复前有确认提示；
7. 恢复后编辑器内容更新；
8. 恢复后刷新页面内容不丢失；
9. 无版本时显示空状态；
10. 获取失败时显示错误提示。
