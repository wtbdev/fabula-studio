# Feature Spec 03：项目剧本编辑工作台

## 1. 功能名称

项目剧本编辑工作台

## 2. 功能目标

实现叙幕工作室 MVP 的沉浸式剧本编辑能力。用户进入 `/projects/:id/edit` 后，不再使用普通后台侧边栏布局，而是进入一个独立工作台：左侧管理剧本场次，中间承担主要剧本文本编辑，右侧展示当前场次的辅助信息、原文依据和后续 AI 建议入口。

工作台强调：

1. 剧本结构清晰；
2. 编辑器是页面主焦点；
3. AI 与拓展功能辅助作者判断，不替代作者编辑；
4. 从小说原文到剧本场次的证据链可逐步展开。

## 3. 功能范围

### 包含

1. 获取项目详情；
2. 获取项目场次列表；
3. 展示左侧场次列表；
4. 默认选中第一场；
5. 切换当前场次；
6. 展示和编辑当前场次内容；
7. 保存当前场次内容；
8. 切换场次前保存当前内容；
9. 顶部工作台工具栏；
10. 右侧拓展信息区；
11. 返回项目列表。

### 不包含

富文本编辑器、CodeMirror、角色编辑器实际编辑能力、设置页实际配置、导出文件生成、场次拖拽排序、新增场次、删除场次、场次合并和拆分、Agent 对话。

## 4. 用户故事

- 作为用户，我希望进入编辑页后看到完整工作台，以便专注打磨剧本。
- 作为用户，我希望左侧可以快速浏览所有场次，以便理解剧本结构。
- 作为用户，我希望中央区域是主要写作区，以便逐场修改剧本。
- 作为用户，我希望右侧看到当前场次摘要、地点、时间和原文线索，以便确认 AI 改编是否可靠。
- 作为用户，我希望离开场次前自动保存修改，以便内容不丢失。

## 5. 页面路由

```txt
/projects/:id/edit
```

该路由需要登录，并使用独立工作台布局，不展示普通后台侧边栏和后台顶栏。

## 6. 页面布局

桌面端使用三栏工作台：

```txt
┌──────────────────────────────────────────────────────────────┐
│ 顶部工具栏：退出编辑 / 模块切换 / 导出 / AI点数 / 用户入口       │
├──────────────┬──────────────────────────────┬────────────────┤
│ 剧本场次区域  │ 编辑器区域                     │ 拓展功能区       │
│ Scene List   │ Script Editor                 │ Info / Source   │
└──────────────┴──────────────────────────────┴────────────────┘
```

推荐宽度：

| 区域 | 宽度 |
|---|---|
| 场次区 | 280px - 320px |
| 编辑器区 | 自适应，占主要空间 |
| 拓展区 | 300px - 360px |

## 7. 顶部工具栏

顶部工具栏固定在工作台顶部，高度约 56px - 64px。

左侧：

1. 退出编辑按钮，返回 `/projects`；
2. 当前模块导航：
   - 剧本编辑器；
   - 角色编辑器；
   - 设置。

右侧：

1. 当前保存状态；
2. 保存按钮；
3. 导出按钮；
4. AI 点数展示；
5. 用户入口。

当前模块需要高亮。剧本编辑器为默认选中模块。角色编辑器和设置在 MVP 中可作为占位入口。

保存状态：

| 状态 | 说明 |
|---|---|
| 已保存 | 当前内容和后端一致 |
| 未保存 | 当前内容有修改 |
| 保存中 | 正在提交保存请求 |
| 保存失败 | 保存接口调用失败 |

## 8. 左侧：剧本场次区域

### 8.1 展示内容

每个场次项展示：

1. 场次序号；
2. 场次标题；
3. 地点；
4. 时间；
5. 简短概要。

示例：

```txt
第 1 场 雨夜重逢
旧车站｜夜晚
林舟与许晚在雨夜重逢。
```

### 8.2 交互要求

1. 页面加载后自动请求场次列表；
2. 默认选中第一场；
3. 点击某一场后切换当前场次；
4. 当前选中的场次需要高亮；
5. 如果当前场次存在未保存修改，切换前先自动保存；
6. 如果保存失败，提示用户并阻止切换。

## 9. 中央：编辑器区域

MVP 阶段使用 Naive UI 的 `n-input type="textarea"`。后续可以替换为 CodeMirror 或结构化剧本编辑器。

展示内容：

1. 当前场次标题；
2. 当前场次地点、时间；
3. 当前场次剧本文本 `content`；
4. 保存状态提示。

交互要求：

1. 用户可以直接编辑文本；
2. 文本发生变化后，保存状态变为“未保存”；
3. 点击保存按钮后调用保存接口；
4. 保存成功后状态变为“已保存”；
5. 保存失败后显示错误提示；
6. 刷新页面后应显示最近一次保存的内容。

## 10. 右侧：拓展功能区

MVP 阶段先实现基础拓展信息面板。

P0 内容：

1. 当前场次摘要；
2. 地点；
3. 时间；
4. 出场人物；
5. 原文依据摘要；
6. 最近更新时间。

P1 内容：

1. AI 修改建议占位；
2. 原文依据预览；
3. 场次问题提示。

P2 内容：

1. 角色信息；
2. 批注；
3. Agent 辅助；
4. 重新生成当前场。

右侧拓展区不能抢占中央编辑器焦点。AI 文案需要强调“建议”和“依据”，避免呈现为聊天机器人主导编辑。

## 11. API 依赖

### 获取项目详情

```http
GET /api/projects/:projectId
```

### 获取项目场次列表

```http
GET /api/projects/:projectId/scenes
```

### 获取单个场次详情

```http
GET /api/scenes/:sceneId
```

### 保存场次内容

```http
PATCH /api/scenes/:sceneId
```

请求体：

```json
{
  "content": "【第 1 场】雨夜车站重逢\n\n地点：废弃车站\n时间：深夜\n\n动作：雨水顺着站台边缘流下。\n\n许晚：你终于肯来了。"
}
```

## 12. 数据结构

```ts
type SaveStatus = 'saved' | 'dirty' | 'saving' | 'failed'

interface Scene {
  id: string
  projectId: string
  sceneNo: number
  title: string
  location?: string
  timeText?: string
  summary?: string
  content: string
  rawJson?: SceneRawJson
  createdAt?: string
  updatedAt: string
}
```

## 13. 前端组件拆分

```txt
views/
  ProjectEditorView.vue

components/
  SceneList.vue
  ScriptEditor.vue
```

### ProjectEditorView.vue

负责工作台状态、项目详情、场次列表、当前场次、保存逻辑、模块导航、右侧拓展区和返回项目列表。

### SceneList.vue

负责展示场次列表、高亮当前场次、触发场次切换事件。

### ScriptEditor.vue

负责接收当前场次内容、展示 textarea、监听文本变化、触发内容更新。

## 14. 响应式规则

桌面端：

1. 三栏同时展示；
2. 顶部工具栏单行展示；
3. 中央编辑器占主要空间。

中等屏幕：

1. 右侧拓展区移到编辑器下方或折叠；
2. 场次区保持在左侧或变为上方横向区域。

移动端：

1. 默认展示编辑器；
2. 场次区和拓展区可堆叠；
3. 不允许横向滚动；
4. 顶部工具栏允许换行。

## 15. 验收标准

1. 用户可以从项目列表进入编辑工作台；
2. 编辑页不展示普通后台侧边栏；
3. 顶部工具栏展示退出编辑、模块导航、保存、导出、AI 点数；
4. 左侧显示场次列表；
5. 页面加载后默认选中第一场；
6. 当前场次在左侧高亮；
7. 中央编辑器可以修改剧本文本；
8. 修改后保存状态显示为未保存；
9. 点击保存按钮可以保存当前场次；
10. 切换场次前可以保存当前场次；
11. 保存失败时阻止切换并提示；
12. 右侧显示当前场次拓展信息；
13. 320px 以上无横向溢出。

## 16. 今日实现优先级

### P0：必须实现

1. 独立工作台布局；
2. 顶部工作台工具栏；
3. 左侧场次列表；
4. 中央 textarea 编辑器；
5. 右侧基础拓展信息；
6. 保存状态；
7. 退出编辑；
8. 切换前保存。

### P1：尽量实现

1. 模块导航占位；
2. 导出按钮占位；
3. 原文依据展示；
4. AI 建议占位；
5. 响应式优化。

### P2：暂不实现

角色编辑器、设置页、CodeMirror、导出文件、拖拽排序、批注和 Agent 辅助。
