# Frontend Feature Spec 06：本场重新生成

## 1. 功能名称

本场重新生成

## 2. 功能目标

允许用户针对当前场次调用 AI 重新生成剧本文本，而不需要重新生成整个项目。用户可以输入改写要求，例如“减少旁白”“增强冲突”“让对白更自然”，AI 返回新的场次版本，用户确认后再应用到编辑器。

该功能用于降低局部修改成本，是剧本编辑器中最重要的 AI 增强功能之一。

---

## 3. 功能范围

### 本阶段实现

1. 在编辑器中提供“重新生成本场”按钮；
2. 用户可以输入改写指令；
3. 生成前扣除固定 AI 点数；
4. AI 返回新版本内容；
5. 弹窗展示原文和新版本；
6. 用户确认后应用新版本；
7. 用户取消后不修改原文；
8. 应用后保存当前场次内容。

### 本阶段不实现

1. 多版本同时生成；
2. 逐段 diff 高亮；
3. 自动判断是否更优；
4. 跨场次重写；
5. 结合完整 RAG 的深度重写。

---

## 4. 页面入口

项目编辑页：

```txt
/projects/:id/edit
```

按钮位置建议：

1. 编辑器顶部工具栏；
2. 或编辑器底部操作区。

按钮文案：

```txt
重新生成本场
```

---

## 5. 交互流程

### 5.1 发起重新生成

1. 用户选中一个场次；
2. 点击“重新生成本场”；
3. 打开弹窗；
4. 用户输入改写要求；
5. 弹窗展示点数提示：

```txt
本次重新生成将消耗 80 AI 点数。
```

6. 用户点击“开始生成”；
7. 调用重新生成接口。

---

### 5.2 结果预览

接口返回后，弹窗显示左右对比：

```txt
左侧：当前版本
右侧：AI 新版本
```

操作按钮：

1. 应用新版本；
2. 重新生成；
3. 取消。

MVP 阶段可以不做逐行 diff，只做双栏文本对比。

---

### 5.3 应用新版本

1. 用户点击“应用新版本”；
2. 将 `regeneratedContent` 写入当前编辑器；
3. 调用保存场次接口；
4. 保存成功后关闭弹窗；
5. 更新保存状态为“已保存”；
6. 更新用户 AI 点数。

---

### 5.4 取消应用

1. 用户点击取消；
2. 弹窗关闭；
3. 当前编辑器内容不变。

---

## 6. 弹窗设计

组件建议：

```txt
components/editor/SceneRegenerateModal.vue
```

Props：

```ts
interface Props {
  visible: boolean
  sceneId: string
  sceneContent: string
}
```

Emits：

```ts
interface Emits {
  close: []
  apply: [content: string]
  refreshUser: []
}
```

---

## 7. 表单字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 改写要求 | textarea | 否 | 用户对本场重新生成的要求 |

快捷指令按钮：

1. 减少旁白；
2. 增强冲突；
3. 让对白更自然；
4. 增加动作描写；
5. 缩短本场内容。

点击快捷指令后，将文字填入改写要求输入框。

---

## 8. API 依赖

```http
POST /api/scenes/:sceneId/regenerate
PATCH /api/scenes/:sceneId
```

## 9. 状态设计

```ts
interface RegenerateState {
  visible: boolean
  loading: boolean

  error: string | null
}
```

---

## 10. 错误处理

| 场景 | 前端提示 |
|---|---|
| 当前内容为空 | 当前场次内容为空，无法重新生成 |
| 点数不足 | AI 点数不足，无法重新生成 |
| 接口失败 | 本场重新生成失败，请稍后重试 |
| 保存失败 | 新版本已生成，但保存失败，请手动保存 |
| 未登录 | 登录状态已失效，请重新登录 |

---

## 11. 验收标准

1. 编辑器页面存在“重新生成本场”入口；
2. 点击后可以打开重新生成弹窗；
3. 用户可以输入改写要求；
4. 生成前会提示点数消耗；
5. 生成时按钮 loading；
6. 生成成功后展示原文和新版本；
7. 用户点击应用后编辑器内容更新；
8. 应用后内容可以保存到后端；
9. 用户点击取消后原文不变；
10. 点数不足时不能发起重新生成。
