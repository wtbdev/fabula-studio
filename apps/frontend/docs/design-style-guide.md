# 叙幕工作室设计风格规范

本文档记录当前前端项目已经形成的整体视觉风格，用于后续页面、组件、原型或跨对话协作时保持一致。

## 设计关键词

- 小清新、文学感、纸张感、可信赖的创作工具。
- 面向小说作者、编剧和内容创作者，不做营销页式的夸张表达。
- 视觉气质应像一份正在被整理的手稿：清爽、克制、有层次，但不冷冰冰。
- 功能呈现强调“小说原文 -> 结构解析 -> 剧本场次 -> AI 建议 -> 可继续编辑”的证据链。

## 核心视觉隐喻

首页第一屏采用“信纸从信封中抽出”的结构：

- Hero 本体是一张信纸，承载主标题、说明和剧本编辑器预览。
- Hero 下方延伸为同一张信纸，进入下一屏的 V 形信封口。
- V 口中间必须露出 hero 信纸延伸，不应被信封前片遮住。
- 信封前片是白色纸面，使用两个向外展开的斜边形成 V 口。
- 阴影应主要加在 hero 到 V 口之间的信纸延伸区域，表达纸张插入信封的层次。
- 信封本身保持干净，只保留轻微纸面阴影和抗锯齿边线。

## 色彩规范

当前 Tailwind theme token 位于 `src/style.css`：

```css
--color-ink: #17211f;
--color-muted: #596966;
--color-paper: #fffdf8;
--color-canvas: #f4f6f4;
--color-line: #dce3df;
--color-sage: #2f7664;
--color-sage-soft: #eef7f3;
--color-brick: #8d4938;
```

使用建议：

- `ink` 用于主文本、标题、品牌文字。
- `muted` 用于说明文本、弱信息、元数据。
- `paper` 用于纸张、卡片、编辑器面板。
- `canvas` 用于页面背景。
- `line` 用于边框和分隔线。
- `sage` 用于主按钮、关键图标、成功态和品牌强调。
- `sage-soft` 用于轻背景、激活态、柔和标签。
- `brick` 用于 AI 建议、证据链、批注等文学手稿感的强调色。

避免：

- 不要把页面做成大面积单一绿色或米黄色。
- 不要使用强紫蓝渐变、深色科技风、重营销风背景。
- 不要添加装饰性渐变球、光斑、bokeh。

## 字体规范

当前字体 token：

```css
--font-ui: "Noto Sans SC", "PingFang SC", "Microsoft YaHei", Inter, ...
--font-display: "Noto Serif SC", "Source Han Serif SC", "Songti SC", ...
--font-script: "Kaiti SC", "STKaiti", KaiTi, "Noto Serif SC", ...
```

使用建议：

- UI、按钮、标签、表单使用 `font-ui`。
- 首页大标题、模块标题、卡片标题优先使用 `font-display`，体现文学和出版感。
- 剧本正文、编辑器预览、手稿内容使用 `font-script`，营造创作文本氛围。
- 字距保持 `letter-spacing: 0`，不要使用负字距。
- 不要随视口宽度线性缩放字体，使用断点和 clamp 控制布局即可。

## 圆角与阴影

整体圆角保持克制：

- 普通卡片、按钮、纸张面板：`8px`。
- 大型页面区块不要做漂浮卡片嵌套。
- 不要卡片套卡片。

当前阴影 token：

```css
--shadow-paper: 0 24px 64px rgba(23, 33, 31, 0.08);
--shadow-paper-deep: 0 42px 78px rgba(23, 33, 31, 0.11);
--shadow-panel: 0 18px 42px rgba(23, 33, 31, 0.08);
```

使用建议：

- 纸张类元素可以使用柔和多层阴影。
- 信封、证明链卡片使用轻阴影即可。
- 重要的 3D/拟物纸张预览可以有更深阴影，但透明度要低。
- 阴影要服务层级，不要让页面显脏。

## 首页 Hero 规范

Hero 需要作为第一屏的主体验，不做传统营销页 split hero。

结构：

- 左侧：标签、主标题、描述、两个主要操作按钮。
- 右侧：倾斜拟物纸张，展示剧本编辑器。
- 下方：hero 信纸延伸进入 V 形信封口。

右侧倾斜卡片建议内容：

- 顶部展示“剧本编辑器”和状态图标。
- 场次标签展示章节、依据百分比。
- 正文展示一段场景标题、动作描述、对白。
- 底部展示 AI 建议或原文依据批注。
- 后续可升级为“剧本编辑器 + 原文依据 + AI 建议”的组合，而不是单纯文本纸。

Hero 文案气质：

- 直接说明产品价值，不要空泛口号。
- 强调“整理成工程”“可继续创作”“证据链清晰”。
- 避免“颠覆行业”“一键生成爆款”等浮夸表达。

## 卡片与信息模块

信息卡片用于展示功能、证据链、流程步骤：

- 每张卡片只表达一个能力点。
- 标题短，说明清楚，不堆术语。
- 图标使用 `lucide-vue-next`，保持线性、简洁。
- 图标容器可使用 `sage-soft` 或 `brick`。
- 卡片 hover 可以轻微上浮，位移控制在 `4px-6px`。

证据链模块强调：

- 原文来源。
- 结构索引。
- AI 修改建议可预览。
- 作者保留判断权。

## 后台与工具页规范

后台页面使用 Naive UI 为主，风格比首页更安静、操作导向：

- 侧边栏深墨色，品牌使用白纸色。
- 内容区背景保持 `canvas`。
- 表格、列表、表单优先使用 Naive UI 组件。
- 自定义 CSS 只补充布局、间距、品牌色和必要状态。
- 操作按钮带 lucide 图标，避免纯文字工具按钮堆叠。

## Tailwind 使用规范

项目已引入 Tailwind CSS v4：

- Vite 插件：`@tailwindcss/vite`
- CSS 导入：`tailwindcss/theme.css` 和 `tailwindcss/utilities.css`
- 当前没有引入 Preflight，避免影响 Naive UI 的默认组件样式。

使用原则：

- Tailwind 负责常规布局、定位、flex/grid、尺寸和简单状态。
- 复杂视觉，例如信纸、信封、3D transform、纹理背景，保留在 CSS class 中。
- 设计 token 放在 `@theme`，不要在新组件里随意新建近似颜色。
- 可以使用 `@apply` 整理稳定的模块样式，但不要把每个复杂选择器都强行改成 Tailwind。

## 动效规范

动效应轻、慢、自然：

- Hero 元素进入使用 fade + translateY。
- 右侧纸张可以轻微浮动，但幅度要小。
- 卡片 hover 上浮即可，不需要复杂弹跳。
- 支持 `prefers-reduced-motion: reduce`，降低动画时长。

避免：

- 大面积闪烁、强视差、持续抢注意力的装饰动画。
- 功能页面中过多滚动入场动效。

## 文案语气

语气应像创作工具，而不是广告：

- 使用“整理、拆解、映射、打磨、继续创作”等词。
- 强调 AI 是辅助创作，不替代作者。
- 多说具体工作流，少说抽象愿景。
- 中文文案保持自然、克制、有一点文学感。

示例：

- “把小说整理成一份能继续创作的剧本工程。”
- “从原文到场次，线索不丢。”
- “真正好用的 AI 改编不是黑盒生成。”

## 响应式规则

- 保证 320px 以上无横向溢出。
- Hero 在窄屏改为单列，剧本纸张预览缩短内容，隐藏次要对白和 AI 建议。
- 固定格式元素要有明确尺寸、宽度限制或 grid track，避免 hover/动态内容引发布局跳动。
- 按钮文字不能溢出，必要时换行或缩短文案。

## 不建议的方向

- 不要改成通用 SaaS 营销页。
- 不要使用过大的 hero 卡片堆叠、圆角胶囊满屏、渐变球背景。
- 不要让视觉变成纯米色手账风，项目仍然是专业创作工具。
- 不要让 AI 建议看起来像聊天机器人主导一切，作者编辑器应是核心。
- 不要使用真实书籍/影视海报作为主视觉，容易偏离“工具产品”。
