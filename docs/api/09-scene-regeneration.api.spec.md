# API Spec 06：本场重新生成

## 1. 功能说明

本场重新生成接口用于针对单个场次调用 AI 重新生成剧本文本。用户可以传入改写要求，AI 基于当前场次内容和项目改编参数返回新版本内容。

该接口需要消耗 AI 点数。

---

## 2. 点数规则

```txt
本场重新生成：80 AI 点数 / 次
```

---

## 3. 接口定义

### 接口

```http
POST /api/scenes/:sceneId/regenerate
```

### 鉴权

需要登录。

### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| sceneId | string | 是 | 场次 ID |

### 请求体

```json
{
  "instruction": "减少旁白，增强人物对白冲突。",
  "mode": "replace"
}
```

字段说明：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| instruction | string | 否 | 空 | 用户改写要求 |
| mode | string | 否 | replace | replace / polish / shorten / expand |

mode 说明：

| mode | 说明 |
|---|---|
| replace | 重新生成整场 |
| polish | 润色当前场次 |
| shorten | 缩短当前场次 |
| expand | 扩写当前场次 |

---

## 4. 处理流程

1. 校验用户登录；
2. 校验场次存在；
3. 校验场次所属项目属于当前用户；
4. 校验用户 AI 点数不少于 80；
5. 读取当前场次 content；
6. 读取项目 config；
7. 扣除 80 点；
8. 调用 AI 生成新版本；
9. 返回原文和新版本；
10. 不直接覆盖场次内容。

注意：该接口只负责生成新版本，不负责应用。前端确认应用后，需要调用 `PATCH /api/scenes/:sceneId` 保存新内容。

## 6. AI Prompt 建议

```txt
你是专业编剧助手。请根据当前剧本场次和用户要求，重新生成该场剧本文本。

要求：
1. 保留原场次的核心剧情；
2. 不要改变场次编号、地点和时间，除非用户明确要求；
3. 根据用户指令调整对白、动作和旁白；
4. 输出纯剧本文本，不要输出解释。

项目改编参数：
{{projectConfig}}

用户改写要求：
{{instruction}}

当前场次：
{{sceneContent}}
```

---

## 7. 与版本历史的关系

如果项目已实现版本历史，建议：

1. 调用本接口时不创建版本；
2. 用户点击“应用新版本”并保存时，由保存场次接口自动创建 `ai_regenerate` 类型版本；
3. 前端保存时可在请求体中传入 `versionSource: 'ai_regenerate'`。

保存示例：

```json
{
  "content": "AI 新版本内容",
  "versionSource": "ai_regenerate"
}
```

---

## 8. 错误码

| code | message | 说明 |
|---:|---|---|
| 40001 | 未登录 | token 不存在或失效 |
| 40301 | 无权访问该场次 | 场次不属于当前用户 |
| 40401 | 场次不存在 | sceneId 无效 |
| 40901 | 当前场次内容为空 | 无法重新生成 |
| 40201 | AI 点数不足 | 点数少于 80 |
| 50012 | 本场重新生成失败 | AI 调用失败 |
